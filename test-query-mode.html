🚀 Query Mode Feature Demo

📋 How to Use Query Mode

  1. Switch to Query Mode in the extension options
  2. Set Base URL (e.g., https://tatain.service-now.com)
  3. Copy Encoded Query from ServiceNow filter builder
  4. Paste Query into the Query Mode textarea
  5. Set Custom Notification Text for each queue

✅ Benefits of Query Mode

  • 📋 Copy-Paste Simplicity: Just copy the encoded query from ServiceNow, no need to construct complex URLs
  • 🔄 Easy Updates: Change query in ServiceNow, copy new query, paste in extension
  • 🎯 Same Functionality: All existing features work exactly the same (notifications, polling, alerts)
  • 🔍 Better Debugging: Copy exact query from ServiceNow filter builder for accurate testing
  • 📱 User-Friendly: Large textareas for easy query editing

🎯 How to Switch Modes

In the extension options page:

  1. Go to Configuration tab
  2. Look for "Input Mode" section
  3. Select Query Mode instead of URL Mode
  4. Fill in your Base URL and paste your queries
  5. Save configuration

🔧 Technical Implementation

The extension automatically:

  • URL-encodes the query parameters
  • Adds required API parameters (sysparm_display_value=true, sysparm_limit=1000)
  • Maintains all existing notification logic
  • Preserves new ticket detection algorithm
  • Works with both single and dual queue configurations
